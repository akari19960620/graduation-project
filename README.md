# graduation-project

# ケアバンク

# # サービス概要
**ケアバンク** は、介護施設を探すご家族が、口コミやコメントを通じて施設の「実際の雰囲気」を知ることができるサービスです。

見学だけではわからない日常の様子や入居者の声を事前に知ることで、入居後のミスマッチを減らし、本人もご家族も安心できる施設選びをサポートします。

# # アイデアが生まれたきっかけ
介護職として働いていた時、「住み慣れた家で過ごしたい」と考える高齢者の方が多かったですが、独居を心配されるご家族も多く話し合った末に施設の入居を検討したり、ご自宅でご家族の力を借りながら生活していたが介護度が高くなり介護負担が大きいためやむなく施設への入居を検討されたりと様々な理由から施設への入居を希望される方が多くいました。
施設はケアマネから数か所紹介してもらい、見学に行ったり施設の空き待ちをしている方が多かったように感じます。

ですが、施設といっても様々で特徴や特色も異なります。
事前の打ち合わせで細かく確認、すり合わせができるといいですが、実際に入居すると「見学時には気にならなかったけど、どうしても合わない。」等が出てきます。
100％マッチする施設を探すのは難しいかもしれませんが、相性のいい施設で穏やかに過ごせるようアプリで何かお手伝いができないかと考えたのがきっかけです。

# # 課題の整理
**高齢者とそのご家族が施設に対する情報が少ない**

- 施設の種類を詳しく知らないため、ケアマネ頼りになっている。
- 取り合えす空き待ちして、入居してしまい後から不満が出てくる
- 比較することができない
- 情報収集、情報共有できる場所が少ない

 ###本当の課題

 **入居する高齢者本人の後悔やご家族の罪悪感**

# # 想定ユーザー
**初めて施設探しをする高齢者のご家族**
- 年齢層は介護職時代に見てきた方々をイメージしたため40-50代で仕事や家事の合間に情報収集をすることを想定。
- ケアマネからの情報＋αで使用する

※高齢者は機械が苦手な方が多いため、メインはご家族。

# # 環境設定
**隙間時間にスマートフォンやiPadで情報収集**
- 忙しい合間をぬってちょこちょこ情報収集ができるように、お気に入り登録ができる機能や閲覧記録を見れるようにする。

# # 既存サービス・競合調査
**サービス名：ケアマップ**
- どんなことができるか：
現在地や地図から周辺施設を探せる。
住所、電話番号、ウェブサイトがすぐに確認できる。
お気に入り登録ができる。

# # 差別化を「体験価値」で表現する
利用者同士で情報共有（コメントでのやり取りや口コミ）ができることで、見学だけではわからなかった実際の雰囲気が入居後の方のコメントや同じ施設を検討中のコメントなどからイメージしやすくなる。

# # 価値を１文で表す
施設探し中の人が、口コミを通じて施設の雰囲気を知ることができる

# # このアプリで実現すること
###MVPで作る機能
　＜トップ画面、質問（診断）画面、結果・施設詳細画面、口コミの5段階評価と自由記述欄＞

###本リリースで作る機能
　＜ユーザー登録・ログイン機能、マイページ、条件絞り込み検索、施設のリンク、お気に入り登録機能、閲覧履歴保存機能、＞

# # このアプリの懸念点、対策
**口コミやコメントの過激化**

**MVP段階での対策**

# # 技術選定理由
**Ruby on Rails**

- MVPに必要な機能が標準機能で実装できる。
- 日本語情報が豊富で詰まったときに調べやすい。

**PostgreSQL**

- MVPのデータ構造を扱うのに十分で、カリキュラムでも学習しているため。

**render**

- 無料枠で始められるため
- PostgreSQLを使用できるため。
